Dotfiles

version control my personal dotfiles

View project onGitHub

Why

It is time to version control and share (with my different machines and whoever else wants to peek)!

How I did set it up

I decided to follow the detached worktree approach. This means that I have somewhere a directory which is the git repository for my dotfiles. This directory is NOT my HOME, there are no symlinks or installation script.

I have it at $HOME/git/dotfiles.git. And I have an alias for git commands when I have to manage it, home, which stands for:

$ alias home
alias home='git --work-tree=/Users/espin --git-dir=/Users/espin/git/dotfiles.git'

So when I want to add a new dotfile in my $HOME, I issue:

$ home add .newdotfile
$ home commit -m "a new dotfile"
